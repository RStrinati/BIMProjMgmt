## ‚ôø Accessibility Standards

### Contrast Requirements (WCAG AA)

All color combinations meet or exceed 4.5:1 contrast ratio:

- **Body Text:** Dark gray (#212121) on white = 11.5:1 ‚úì
- **Button Text:** White on blue (#2196F3) = 6.8:1 ‚úì
- **Secondary Text:** Gray (#616161) on white = 4.5:1 ‚úì

### Color Usage Guidelines

‚úÖ **Always use multiple signals:**
- Color + Text label
- Color + Icon
- Color + Pattern (not just color alone)

‚úÖ **Semantic meaning:**
- Green = success/good
- Red = error/bad
- Yellow = warning/caution
- Blue = information

## üåô Dark Mode

Dark mode uses lighter colors on dark backgrounds for reduced eye strain:

- **Background:** #121212 (near black)
- **Paper:** #1E1E1E (dark gray)
- **Text:** #FFFFFF (white, 87% opacity)
- **Text Secondary:** #FFFFFF (60% opacity)
- **Accent Colors:** Lightened for visibility

### When to Use Dark Mode

‚úÖ **Good for:**
- Evening/night use
- Reduced eye strain during long sessions
- OLED displays (saves battery)

‚ùå **Not ideal for:**
- Bright environments
- Print materials
- Accessibility for some visual impairments

## üéØ Best Practices for Designers

### Color Selection

1. **Start with semantic meaning**
   - What status/action does this convey?
   - What emotion should it evoke?

2. **Check contrast ratios**
   - Use WebAIM contrast checker
   - Aim for WCAG AA minimum (4.5:1)

3. **Consider colorblind users**
   - Don't rely on red-green alone
   - Use different shades and icons

4. **Test in light and dark modes**
   - Does it work in both modes?
   - Is contrast still sufficient?

### Typography

1. **Respect hierarchy**
   - One H1 per page
   - Use H2-H6 for structure
   - Don't skip heading levels

2. **Keep line length readable**
   - 50-75 characters for optimal reading
   - Max 120 characters for wider screens

3. **Maintain consistent spacing**
   - Use 8px grid system
   - Consistent gaps between sections

4. **Use weight for emphasis**
   - 700 (bold) for headings
   - 500 (semibold) for labels
   - 400 (regular) for body text

### Component Design

1. **Consistent border radius**
   - Large cards: 12px
   - Buttons/inputs: 6px
   - Small elements: 4px

2. **Proper shadow usage**
   - Elevation 1: 2px shadow
   - Elevation 2: 4px shadow
   - Elevation 3: 8px shadow

3. **Adequate padding**
   - Cards: 16px internal
   - Buttons: 8px vertical, 16px horizontal
   - Input fields: 12px padding

4. **Focus states**
   - Always visible (minimum 2px outline)
   - Use primary color
   - Test with keyboard navigation
